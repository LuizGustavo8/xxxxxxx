Artefatos: Sua Fábrica de Entregáveis Visuais

O que é

Código que roda na conversa

Quando você pede algo visual, o Claude gera código (HTML, React/JSX, SVG ou Mermaid) que é compilado e renderizado em tempo real como uma preview interativa dentro da própria conversa. Não é uma imagem — é código real rodando. Você pode clicar, digitar, interagir.

Como acessar

Nativo no claude.ai

Vá em Configurações → Feature Preview → Artifacts e ative. A partir daí, basta pedir algo que tenha saída visual e o Claude cria o artefato automaticamente. O resultado aparece em um painel lateral que você pode expandir, copiar o código, ou baixar o arquivo.

Para que usar

Tudo que é visual e auto-contido

Landing pages, dashboards interativos, calculadoras, formulários, diagramas, protótipos de UI, email templates HTML, media kits, one-pagers, jogos, ferramentas internas — qualquer coisa que seja front-end puro sem backend.

!

Dica crucial: Artefatos React podem usar bibliotecas como Recharts, D3, Chart.js, Three.js, Tailwind CSS e shadcn/ui — sem instalar nada. Peça interatividade e o Claude usa hooks do React. Peça persistência de dados e ele usa a API window.storage que salva entre sessões.

Skills

Skills: Arquivos Profissionais sob Demanda

O que é

Instruções de produção de arquivos

Skills são conjuntos de melhores práticas que o Claude consulta antes de gerar um arquivo. Quando você pede "crie uma apresentação", ele lê internamente o skill de PPTX que contém regras sobre hierarquia de slides, tipografia, uso de cores e estrutura — como um designer experiente seguindo um styleguide.

Como acessar

Ativação automática

Ative Configurações → Feature Preview → "Code Execution and File Creation". A partir daí, basta pedir qualquer arquivo: "crie uma apresentação sobre X", "monte uma planilha de Y", "gere um relatório em Word de Z". O Claude detecta o formato e aciona o skill correto automaticamente. Você recebe um link de download.

Formatos disponíveis

4 tipos de saída

PPTX — Decks completos com slides formatados, speaker notes e hierarquia visual.
DOCX — Propostas, relatórios, contratos com TOC, cabeçalhos e estilos profissionais.
XLSX — Planilhas com fórmulas, formatação condicional e gráficos.
PDF — Documentos finalizados, certificados, infográficos.

!

Skill customizado: Você pode criar seus próprios skills (via Claude Code ou documentação Cookbook). Imagine um skill que sempre gera propostas no template visual da sua empresa, com logo, cores e estrutura padronizados. Pesquise "Skills Notebooks" na documentação oficial para aprender a criar.

Conectores MCP

Conectores: O Claude que Executa

O que é

Integração via MCP

Conectores usam o Model Context Protocol (MCP) — um protocolo aberto criado pela Anthropic para que IAs interajam com ferramentas externas. O Claude pode ler, criar e modificar dados diretamente em serviços como Gmail, Google Calendar, Notion, Jira e HubSpot. Não é só consulta: é ação bidirecional.

Como configurar

3 cliques para conectar

No claude.ai, vá em Configurações → Connectors. Você verá a lista de integrações disponíveis. Clique em "Connect" ao lado do serviço desejado, autentique com sua conta (OAuth), e pronto. A partir desse momento, basta pedir ao Claude em linguagem natural: "veja minha agenda de amanhã" ou "busque emails sobre o projeto X".

Conectores disponíveis

14+ integrações nativas

Gmail Google Calendar Notion Linear Asana Jira / Confluence HubSpot Microsoft 365 Monday.com Figma Box Canva Intercom

Projetos

Projetos: Contexto que Nunca se Perde

O que é

Espaço com memória definida

Um Projeto é um espaço no Claude onde você define instruções personalizadas (system prompt) e anexa arquivos de referência. Toda conversa criada dentro do projeto herda essas instruções e tem acesso aos arquivos — sem precisar re-enviar ou re-explicar a cada vez.

Como configurar

Passo-a-passo

No claude.ai, clique em "Projects" na barra lateral → "Create Project". Defina: (1) o nome, (2) as instruções personalizadas no campo de texto (tom, regras, framework), (3) anexe até 200k tokens em arquivos de referência (PDFs, docs, planilhas, código). Cada nova conversa dentro desse projeto obedece essas configurações.

Quando usar

Trabalho recorrente

Use quando precisa que o Claude sempre siga um padrão específico: tom de voz da marca, framework de consultoria, tabela de preços, template de proposta, guidelines de conteúdo. Ideal para trabalhar com o mesmo cliente, produto ou tema por semanas ou meses.

!

Projetos vs. Memória: A Memória do Claude salva preferências gerais (ex: "prefiro respostas curtas"). Projetos são contextos segmentados com escopo definido — o Claude dentro de um Projeto "Cliente ACME" não carrega informações do Projeto "Marketing". É isolamento intencional.

Cowork

Cowork: O Agente Autônomo no Desktop

O que é

Trabalho de conhecimento agêntico

Cowork é um app desktop para não-desenvolvedores onde o Claude trabalha de forma autônoma em tarefas complexas. Ele navega na web, pesquisa, analisa documentos e gera entregáveis — tudo em sequência, sem supervisão. Pense nele como um analista competente que recebe um briefing e volta com o trabalho pronto.

Como acessar

App desktop + mobile

Disponível como aplicativo desktop (Windows/Mac). Também pode ser iniciado e monitorado remotamente pelo app mobile do Claude. O Cowork utiliza as extensões de Excel, PowerPoint e Chrome como ferramentas internas — ele abre planilhas, edita slides e navega em sites por conta própria.

Quando usar

Tarefas multi-etapa demoradas

Pesquisa profunda cruzando múltiplas fontes. Análise de documentos extensos. Montagem de relatórios combinando dados diferentes. Projetos que no chat levariam várias conversas — no Cowork, você define o objetivo e ele executa autonomamente.

!

Cowork vs. Chat: Use o chat quando o trabalho é conversacional e iterativo (brainstorm, revisar, ajustar). Use o Cowork quando o trabalho é delegável com um briefing claro (pesquisa, compilação, análise comparativa). A diferença é o nível de autonomia.

Claude Code

Claude Code: Desenvolvendo no Terminal

O que é

Agente de código na CLI

Claude Code é uma ferramenta de linha de comando que permite delegar tarefas de código direto do terminal, do app desktop ou do celular. Diferente dos artefatos, o Claude Code trabalha no seu repositório real: lê arquivos, entende a codebase, executa comandos, roda testes e faz commits.

Como instalar

Requer Node.js

Pré-requisito: Node.js instalado. Instalação:
npm install -g @anthropic-ai/claude-code

Depois: claude no terminal para iniciar. Autentique com sua conta claude.ai (plano Pro+) ou com API key. Acessível pelo app desktop e mobile remotamente.

Quando usar

Código em produção

Refatorar código existente. Implementar features em projetos já iniciados. Debugging — lê logs, identifica causa, aplica correção. Gerar testes. Setup de projetos. Também funciona com servidores MCP para interagir com APIs externas.

!

Combinação poderosa: Prototipe no chat com artefatos (UI perfeita grátis) → Exporte o código → Use Claude Code para integrar no projeto real com backend, testes e deploy. É assim que você substitui o Lovable em 90% dos casos.

Boas práticas

Prompts que Geram Resultados Melhores

C

Contexto

Quem é você, para quem é, qual o objetivo. "Sou consultor preparando deliverable para o board do cliente."

T

Tarefa

O que exatamente produzir. "Crie um deck de 10 slides sobre tendências de IA no varejo."

F

Formato

Tipo de saída, extensão, estrutura. "PPTX com visual executivo, max 5 bullets por slide."

R

Referência

Estilo ou benchmark. "No estilo McKinsey — frameworks, dados, insights acionáveis."

E

Exclusões

O que evitar. "Sem jargão técnico. Não cite concorrentes. Valores em BRL."

Prompt ruim ❌
Faz uma apresentação sobre IA.
Prompt bom ✓
CONTEXTO: Consultor de inovação, apresentação para board de varejo com 200 lojas.

TAREFA: Deck sobre IA otimizando operações de loja, CX e supply chain.

FORMATO: PPTX, 10 slides, visual executivo e clean.

REFERÊNCIA: Estilo McKinsey — dados, frameworks visuais, insights acionáveis.

EXCLUSÕES: Sem jargão técnico. Max 5 bullets/slide. Valores em BRL.
→

Para iterar: Seja cirúrgico. "No slide 4, troque barras por linha com tendência 12 meses." Edições específicas > refazer tudo.

Implementação

Checklist: 4 Semanas para Dominar

Semana 1 — Setup

Fundação do ambiente

Ative Artifacts e File Creation nas Feature Previews. Conecte 3 ferramentas em Connectors (Calendar, Gmail, CRM/PM tool). Configure preferências pessoais em Settings → User Preferences. Crie seu primeiro Projeto para o cliente ou workstream principal.

Semana 2 — Primeiras entregas

Substituir o primeiro fluxo manual

Gere sua primeira proposta ou relatório com Skills (DOCX). Crie um artefato React útil (calculadora, dashboard, comparativo). Use conectores para automatizar algo manual (resumo de emails, preparação de reunião).

Semana 3 — Combinar

Fluxos integrados

Dentro de um Projeto configurado: use conectores para puxar dados → gere dashboard como artefato → produza relatório DOCX → transforme em deck PPTX. Tudo na mesma sessão. Experimente o Cowork para pesquisa profunda.

Semana 4 — Escala

Documentar e multiplicar

Documente melhores prompts num Projeto "Playbook Interno". Treine colegas nos fluxos validados. Identifique ferramentas pagas para substituir. Se relevante: instale Claude Code e explore prototipação → produção.
